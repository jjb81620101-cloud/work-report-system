🚀 雲端部署教學

將工作回報系統部署到網路上，讓同事用手機行動網路也能填寫。

三個服務全部免費：
免費GitHub — 存放程式碼 ・ 免費Render — 執行伺服器 ・ 免費Cloudinary — 儲存照片

① 把程式碼上傳到 GitHub

1
前往 github.com 登入（或免費註冊）
2
點右上角 + → New repository
名稱填 work-report-system，選 Private（私人），建立
3
打開 PowerShell，切到你的資料夾： 
cd "$env:USERPROFILE\Documents\Claude\Projects\線上工作回報系統"
4
依序執行（把 你的帳號 換成你的 GitHub 帳號名稱）： 
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/你的帳號/work-report-system.git
git push -u origin main
⚠️ credentials.json 已在 .gitignore 裡，不會被上傳，放心。

② 照片存在 Google Drive（不用額外設定）

✅ 照片會自動存到你 Google Drive 的「工作回報照片」資料夾，依年月/日期分層整理。
只要 credentials.json 已設定好，Drive 照片上傳完全自動，不需要額外帳號。

③ 設定 Google Sheets（如果還沒設定）

如果你已經在本機設定好 credentials.json，直接跳到下一步。
依照 設定說明.html 的第三步完成 Google Sheets 設定，取得 credentials.json。

然後用文字編輯器（記事本）打開 credentials.json，複製全部內容備用（下一步要貼進 Render）。

④ 在 Render 部署伺服器

1
前往 render.com 免費註冊，用 GitHub 帳號登入
2
點 New + → Web Service
3
選 Connect a repository，連結你的 work-report-system
4
填寫以下設定：
欄位填入值
Namework-report（任意）
RuntimePython 3
Build Commandpip install -r requirements.txt
Start Commandpython server.py
Instance TypeFree
5
往下找到 Environment Variables，新增以下四個變數：
Key（名稱）Value（值）
GOOGLE_CREDENTIALScredentials.json 的全部內容（整個 JSON 貼上）
6
點 Create Web Service，等待 2~3 分鐘部署完成
✅ 部署完成後，Render 會給你一個網址，例如：
https://work-report.onrender.com

把這個網址傳給同事，任何網路、任何手機都能使用！

⑤ 注意事項

免費版會休眠：Render 免費方案在 15 分鐘沒人使用後會進入休眠，下次有人開啟時需要等約 30 秒才會醒來。這是正常現象，不影響資料安全。

如何更新程式碼？

只要在 PowerShell 執行： 
git add .
git commit -m "update"
git push
Render 會自動偵測到並重新部署，約 2 分鐘完成。

Google Sheets 試算表名稱

試算表必須命名為「工作回報系統」（完全一致），且已將服務帳號 email 加入共用（編輯者權限）。
