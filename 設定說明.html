📋 工作回報系統 — 設定與使用說明

系統架構：Python Flask 後端 + SQLite 本地資料庫 + Google Sheets 自動同步。
所有資料儲存在筆電上，Google Sheets 為同步副本，依日期自動開分頁。

📁 資料夾結構

線上工作回報系統/
  ├── server.py # Flask 後端主程式
  ├── requirements.txt # Python 套件清單
  ├── start.sh # 一鍵啟動腳本（Mac/Linux）
  ├── credentials.json # Google 服務帳號金鑰（你要自己放）
  ├── reports.db # SQLite 資料庫（自動建立）
  ├── templates/
  │   ├── index.html # 員工填寫表單
  │   └── admin.html # 後台管理頁面
  └── uploads/ # 上傳的照片存放處

第一步：確認 Python 已安裝

打開終端機（Terminal），輸入： 
python3 --version
若看到 Python 3.10.x 以上即可。
若尚未安裝，請到 python.org 下載並安裝。

第二步：啟動伺服器

1 打開終端機，切換到系統資料夾： 
cd /path/to/線上工作回報系統
2 執行啟動腳本（會自動安裝套件並啟動）： 
bash start.sh
看到以下訊息即代表成功： 
  線上工作回報系統 啟動中...
  前台（填寫表單）：http://localhost:5000
  後台（查看回報）：http://localhost:5000/admin
讓同事可以連線：啟動後查詢你的筆電 IP（在終端機輸入 ipconfig getifaddr en0），告訴同事用 http://你的IP:5000 連線，需確保大家在同一個 Wi-Fi 網路內。

第三步：設定 Google Sheets 自動同步

1 建立 Google Cloud 專案

  1. 前往 console.cloud.google.com
  2. 點選「建立專案」，隨意命名（例如：工作回報）
  3. 建立完成後，確認左上角已選到這個專案

2 啟用 API

  1. 左側選單 → API 和服務 → 資料庫
  2. 搜尋並啟用 「Google Sheets API」
  3. 搜尋並啟用 「Google Drive API」

3 建立服務帳號

  1. 左側選單 → API 和服務 → 憑證
  2. 點「建立憑證」→「服務帳號」
  3. 填入名稱（例如：工作回報同步），建立完成
  4. 點入剛建立的服務帳號 → 「金鑰」頁籤
  5. 「新增金鑰」→「建立新的金鑰」→ 選 JSON → 下載
  6. 將下載的 JSON 檔改名為 credentials.json，放到系統資料夾根目錄

4 取得服務帳號 Email

打開 credentials.json，找到 "client_email" 欄位，複製那串 email（格式類似 xxxxx@xxxx.iam.gserviceaccount.com）。

5 建立 Google 試算表並共享

  1. 前往 sheets.google.com，新建一個試算表
  2. 將試算表命名為 「工作回報系統」（需完全一致）
  3. 點右上角「共用」→ 貼上剛才複製的服務帳號 email
  4. 權限選「編輯者」→ 確認共用
完成！重新啟動伺服器後，每次有人提交回報，系統就會自動在 Google Sheets 裡建立對應日期的分頁並寫入資料。後台管理頁面右上角會顯示「Google Sheets 已連線」。

使用說明

頁面網址用途
填寫表單http://IP:5000/員工填寫工作回報
後台管理http://IP:5000/admin依日期查看所有回報與照片

後台功能說明

  • 左側日期列表：顯示所有有回報的日期，點擊切換
  • 每個日期是獨立頁面，不會全部混在一起
  • 照片可點擊放大檢視
  • 右上角顯示 Google Sheets 連線狀態
  • 統計卡片顯示總回報數、天數、最新日期

常見問題

Q：同事無法連線？

確認：① 大家在同一 Wi-Fi ② 用正確 IP（不是 localhost）③ Mac 防火牆是否擋住 port 5000

Q：Google Sheets 沒有更新？

確認：① credentials.json 在正確位置 ② 試算表名稱完全是「工作回報系統」③ 服務帳號 email 有被加入試算表共用

Q：關閉終端機後系統就停了？

可以用 nohup bash start.sh & 讓它在背景執行，或安裝 pm2（npm install -g pm2，然後 pm2 start server.py --interpreter python3）。
